규칙 ID이름설명
C1VARCHAR 바이트 초과컬럼에 정의된 최대 바이트를 넘어서는 값을 찾는다. KODATA 규약: 한글 1자 = 2 byte, 영문/숫자/공백/특수문자 = 1 byte (CP949 기준). 적재 대상 스키마의 max_bytes 를 기준으로 비교한다.
C2값 범위 초과숫자 컬럼의 범위 위반을 찾는다. 지분율(ratio): 0 ~ 100 / 이자율(interest_rate): 0 ~ 9,999.999 / 금액(amount): 정수부 17자리 이하. 숫자로 파싱 실패하면 warning.
C3허용 값(enum) 위반컬럼에 사전 정의된 허용 값 목록이 있을 때, 그 바깥의 값을 전부 오류로 잡는다. 예: data_period 는 ["당기","전기"] 만 허용. 빈 값/NULL 은 C4 가 담당하므로 여기선 넘어간다.
C4필수키 결측반드시 값이 있어야 하는 식별자 컬럼이 NULL / 빈 문자열 / 공백만 / "NULL" 문자열 인 경우. 스키마의 key=Y 컬럼과 기본 필수키(corp_no, business_registration_no, row_group_id 등)에 적용.
C5-1인코딩 깨짐정상 한글이라면 조합형 음절(가~힣) 범위여야 하는데, 한글 자모(ㄱ~ㅎ 초성) 나 박스 드로잉 문자(┐, ━) 가 섞여있는 경우. HTML 변환/OCR 오류 징후.
C5-2자모 전용 셀값 전체가 한글 자모(ㄱ~ㅎ, ㅏ~ㅣ)로만 구성된 경우. 예: "ㄱㅇ", "ㅇㅇㅇ". 정상 입력이 아니라 타자 오류/임시 값.
C5-3impossible 음절한국어 위키 2,171,086 기사에서 빈도가 임계치(기본 1,000회) 미만인 음절이 한 개라도 등장하면 오류. 한글 11,172 음절 중 상위 약 1,500개(=common_syllables) 만 허용하는 셈. KODATA 도메인(감사/공시 문서)은 정형화된 기업 문서라 구어체/의성어 FP가 없어 공격적 임계치 사용. 모든 텍스트 컬럼에 적용 — 회사명/자유서술 구분 없음.
C5-5특수문자 전용값 전체가 특수문자/공백/괄호만으로 구성된 경우(예: "---", "( )"), 또는 non-breaking hyphen("‑") / bullet("•") 가 3회 이상 연속된 경우.
C5-6HTML 엔티티 미변환`&nbsp;`, `&amp;`, `&lt;`, `&gt;`, `&quot;`, `&#숫자;` 등 HTML 엔티티가 디코딩되지 않고 원본 그대로 남아있는 경우. 크롤링/파싱 단계 버그.
C5-7제어문자값에 ASCII 제어문자(\x00~\x1f, \x7f~\x9f 등)가 포함된 경우. 일반 텍스트에는 들어갈 수 없는 문자이므로 명백한 버그.
C6파싱 오류① maturity_date 날짜 포맷 불일치 (YYYY-MM-DD, YYYYMMDD 등 허용) ② 주석 마크만 있는 셀 ("(주)", "(주1)", "*주2" 등 내용 없음) ③ raw_* 와 calculated_* 금액 컬럼이 같은 행에서 0.1% 이상 차이.
